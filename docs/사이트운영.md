# PAINS 콘텐츠 운영 가이드

목표는 명확합니다. **디자인은 코드에 두고, 콘텐츠와 사진은 Google Sheets/Drive에서 관리합니다.**

운영진은 평소에 Codespace를 열지 않습니다. Google Sheets를 수정하고 새로고침하면 사이트가 바뀝니다.

## 실제 운영 흐름

1. Google Sheets에서 문구, 연표, 조직도, 소모임, 이벤트, 이미지 URL을 수정합니다.
2. Google Drive 또는 외부 이미지 저장소에 사진을 올리고, 공유 링크를 시트에 붙입니다.
3. 사이트를 새로고침합니다.

배포는 필요 없습니다. 디자인이나 레이아웃을 바꿀 때만 코드를 수정하고 배포합니다.

## 연결 방식

사이트는 `/api/content.js`를 호출합니다. `/api/content`도 같은 API로 연결되도록 설정해둡니다.

콘텐츠 API는 아래 둘 중 하나로 콘텐츠를 가져옵니다.

- `PAINS_CONTENT_API_URL`: Google Apps Script 웹 앱이 반환하는 JSON
- `PAINS_SHEET_ID`: 공개 Google Sheet의 탭을 직접 읽어서 JSON으로 변환

둘 다 없거나 실패하면 `data/site-content.json`이 fallback으로 사용됩니다.

## 추천 방식

가장 운영하기 쉬운 방식은 **Google Apps Script 웹 앱 방식**입니다.

권장 이름:

```text
PAINS_SITE_CMS
```

권장 Drive 폴더:

```text
PAINS_SITE_ASSETS
```

처음 세팅:

1. Google Drive에서 새 Google Sheet를 만듭니다.
2. 시트 이름을 `PAINS_SITE_CMS`로 바꿉니다.
3. 메뉴에서 `확장 프로그램 > Apps Script`를 엽니다.
4. `google-apps-script/content-api.gs` 전체 내용을 붙여넣고 저장합니다.
5. Apps Script 상단 함수 선택 메뉴에서 `setupPainsCms`를 고르고 실행합니다.
6. 권한 승인 창이 뜨면 승인합니다.
7. 다시 Google Sheet로 돌아오면 `copy`, `home_timeline`, `organization`, `societies`, `events` 등의 탭이 자동 생성됩니다.
8. Apps Script에서 `배포 > 새 배포 > 웹 앱`을 선택합니다.
9. `실행 계정`은 `나`, `액세스 권한`은 `링크가 있는 모든 사용자` 또는 `Anyone`으로 둡니다.
10. 배포 후 나온 Web app URL을 Vercel 환경변수 `PAINS_CONTENT_API_URL`에 넣습니다.

이후부터는 Google Sheets/Drive만 수정하면 됩니다.

이미 세팅된 뒤에는 `setupPainsCms`를 다시 실행하지 마세요. 다시 실행하면 현재 시트 내용이 초기 템플릿으로 덮입니다.

## Vercel 환경변수

Vercel Dashboard에서 설정합니다.

권장:

```text
PAINS_CONTENT_API_URL=https://script.google.com/macros/s/....../exec
```

대안:

```text
PAINS_SHEET_ID=구글시트_ID
```

`PAINS_CONTENT_API_URL`이 있으면 그 값을 우선 사용합니다.

## Drive 이미지 운영

이미지는 Google Drive에 올린 뒤 “링크가 있는 모든 사용자 보기 가능”으로 설정합니다.

시트에는 아래 둘 중 아무거나 넣어도 됩니다.

```text
https://drive.google.com/file/d/FILE_ID/view?usp=sharing
```

```text
https://drive.google.com/uc?export=view&id=FILE_ID
```

사이트 로더가 Drive 공유 링크를 이미지 표시용 URL로 변환합니다.

## 아카이브 PDF 운영

프로젝트와 공지 PDF는 GitHub Release 대신 기존 `PAINS_SITE_CMS` Google Sheet와 Google Drive에서 함께 운영할 수 있습니다.

권장 흐름:

1. Google Drive에 프로젝트/공지 PDF를 올립니다.
2. 파일 공유를 `링크가 있는 모든 사용자 보기 가능`으로 바꿉니다.
3. 기존 CMS Google Sheet에 `projects`, `notices` 탭을 추가하고 메타데이터와 Drive 링크를 입력합니다.
4. 기존 `google-apps-script/content-api.gs`를 Apps Script에 다시 붙여넣고 새 버전으로 배포합니다.
5. `settings.projectArchiveApiUrl`, `settings.noticeArchiveApiUrl`은 비워둡니다. 비어 있으면 `/api/content.js` 안의 데이터를 바로 사용합니다.

`projects` 탭 권장 컬럼:

| title | year | generation | period | sport | driveUrl | driveId | fileName | visible | order |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 야구 특화 LLM: PA-LLMa | 2026 | 10기 | 방학 중 프로젝트 | 야구 | Drive 공유 링크 |  | 10V_10.pdf | TRUE | 1 |

`notices` 탭 권장 컬럼:

| title | date | generation | department | driveUrl | driveId | fileName | important | visible | order |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 고려대학교 스포츠 통계 동아리 PAINS 회칙 | 2026-03-16 | 11기 | 운영위원회 | Drive 공유 링크 |  | Laws_20260316.pdf | TRUE | TRUE | 1 |

`driveUrl`에는 아래 형식의 공유 링크를 넣으면 됩니다.

```text
https://drive.google.com/file/d/FILE_ID/view?usp=sharing
```

`driveId`만 넣어도 됩니다. 둘 다 있으면 `driveId`를 우선 사용합니다.

기존 GitHub Release 아카이브를 임시 fallback으로 유지하고 싶으면 `settings.projectArchiveApiUrl`이나 `settings.noticeArchiveApiUrl`에 별도 아카이브 API URL을 넣어둘 수 있습니다.

| key | value |
| --- | --- |

| resultEndAt | 2026-03-16 23:59 |

| resultHref | result |
| resultApiUrl | 결과 조회 Apps Script Web app URL |

지원/결과 조회 메뉴는 `settings`에서 관리합니다.

- 항상 열기: `applyEnabled` 또는 `resultEnabled`를 `TRUE`
- 항상 닫기: `FALSE`
- 일정으로 자동 관리: `AUTO`로 두고 `StartAt`, `EndAt` 수정
- 날짜 형식: `YYYY-MM-DD HH:mm`, 한국시간 기준

`page_content`

위 탭들로 관리하지 않는 페이지의 문구/이미지를 범용으로 덮어쓸 때 씁니다.

| page | selector | type | value | visible | order |
| --- | --- | --- | --- | --- | --- |
| apply | h2 | text | PAINS 11기 지원하기 | TRUE | 1 |

- `background`: 배경 이미지 교체
- `value`: input 값 교체

## 지금 외부 관리 가능한 영역

- 메인 히어로 문구와 이미지
- 연표
- Who We Are 문구
- About / Projects / Community 카드
- 일정 소개 문구
- 소개 페이지 문구
- 회장 인사말과 이미지
- 조직도
- 소모임
- 이벤트
- 프로젝트 아카이브 API URL
- 기타 페이지의 문구/이미지/링크: `page_content` 탭으로 관리

## 코드가 필요한 경우

- 레이아웃 변경
- 애니메이션 변경
- 새 페이지/새 섹션 추가
- 카드 디자인 변경

문구, 사진, 순서, 노출 여부 변경에는 코드가 필요 없습니다.
